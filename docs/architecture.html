Decyzje warte odnotowania

  • Ten sam obraz, dwie role. z-alpha-backend uruchamiany w pod-zie user-api jako uvicorn user_api.main:app oraz w pod-zie backend jako uvicorn api.main:app. Wspólny kod auth, modeli i walidacji bez duplikacji.
  • host.docker.internal w pod-ach. Docker Desktop wstrzykuje DNS rewrite w CoreDNS, dzięki czemu backend pod woła TWS na hoście bez hostNetwork.
  • LoadBalancer = localhost. Docker Desktop auto-mapuje LB Services na host. Na innym K8s wymagałby MetalLB albo cloud provider.
  • Per-run image tag. start_all.py generuje r{timestamp} i robi kubectl set image — stały tag :local jest pułapką, kubelet nie wie że content się zmienił.
  • Cache prefetch po loginie. Frontend strzela do 10 endpointów analitycznych przy mount-cie dashboardu — backend TTLCache (5 min) napełnia się w tle, użytkownik nie czeka na pierwsze przejście między widokami.
